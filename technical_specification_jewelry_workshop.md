# Техническое задание

## Информационная система «Ювелирная мастерская»

------------------------------------------------------------------------

## 1. Общая характеристика системы

### 1.1 Назначение системы

Информационная система предназначена для автоматизации процесса
оформления и учета заказов на изготовление ювелирных изделий в
мастерской.

### 1.2 Цель разработки

Создание веб-приложения, обеспечивающего: - учет клиентов - учет
заказов - управление материалами - управление типами изделий - расчет
стоимости изделия - хранение информации о выполненных заказах

------------------------------------------------------------------------

## 2. Архитектура системы

Система реализуется по клиент-серверной архитектуре:

-   **Frontend:** React JS (SPA)
-   **Backend:** Node.js + Express
-   **База данных:** MySQL
-   **API:** RESTful
-   Формат обмена данными: JSON

------------------------------------------------------------------------

## 3. Функциональные требования

### 3.1 Управление клиентами

Система должна обеспечивать: - добавление клиента - редактирование
клиента - удаление клиента - просмотр списка клиентов - просмотр истории
заказов клиента

**Поля клиента:** - id - фамилия - имя - отчество - телефон - email -
дата регистрации

------------------------------------------------------------------------

### 3.2 Управление типами изделий

Система должна обеспечивать: - добавление типа изделия - редактирование
типа - удаление типа - просмотр списка типов

**Примеры типов изделий:** - серьги - кольца - браслеты - броши

------------------------------------------------------------------------

### 3.3 Управление материалами

Система должна обеспечивать: - добавление материала - редактирование
материала - удаление материала - просмотр списка материалов

**Поля материала:** - id - название (золото, серебро, платина и т.д.) -
цена за грамм

------------------------------------------------------------------------

### 3.4 Управление заказами

Система должна обеспечивать: - создание заказа - редактирование заказа -
удаление заказа - просмотр списка заказов - фильтрацию заказов по
статусу - просмотр детальной информации по заказу

**Поля заказа:** - id - client_id (FK) - product_type_id (FK) -
material_id (FK) - вес изделия (граммы) - стоимость работы - итоговая
стоимость (рассчитывается автоматически) - статус (новый / в работе /
готов / выдан) - дата создания - дата завершения

------------------------------------------------------------------------

## 4. Бизнес-логика расчета стоимости

Итоговая стоимость рассчитывается по формуле:

    Стоимость = (вес × цена_за_грамм) + стоимость_работы

Расчет выполняется на backend.

------------------------------------------------------------------------

## 5. Нефункциональные требования

-   Аутентификация администратора (JWT)
-   Валидация входных данных
-   Защита от SQL-инъекций
-   Обработка ошибок
-   REST API архитектура
-   Адаптивный интерфейс

------------------------------------------------------------------------

## 6. Структура базы данных

### Таблица clients

-   id (PK)
-   last_name
-   first_name
-   middle_name
-   phone
-   email
-   created_at

### Таблица product_types

-   id (PK)
-   name

### Таблица materials

-   id (PK)
-   name
-   price_per_gram

### Таблица orders

-   id (PK)
-   client_id (FK)
-   product_type_id (FK)
-   material_id (FK)
-   weight
-   work_price
-   total_price
-   status
-   created_at
-   completed_at

------------------------------------------------------------------------

------------------------------------------------------------------------

## 8. Ожидаемый результат

Разработанная веб-система должна позволять: - автоматизировать работу
ювелирной мастерской - вести учет клиентов - оформлять и контролировать
заказы - автоматически рассчитывать стоимость изделий - хранить данные в
базе MySQL
